## Admin UI Upgrade (Release D)

Цель: форма модерации не должна выглядеть как дефолтная форма браузера.

Требования:
- Для каждой заявки: отдельная “карточка модерации” с секциями:
  1) Исходная идея пользователя (read-only)
  2) Поле "Утвержденная формулировка" (textarea в стиле приложения)
  3) Поле "Комментарий модератора" (textarea в стиле приложения)
  4) Действия: Approve / Reject (понятные primary/secondary кнопки)
- Textarea:
  - нормальная рамка/фон/скругления
  - видимый label
  - подсказка под полем (helper text)
  - состояние focus

- Никаких изменений API и бизнес-логики.
